# 前言

一直以来，我都希望有一个真正顺手的图床工具：既可以平时自己用时，最好不要依赖后端，打开就能传，又想做细一点时，又能有仓库管理、命名规则、水印、压缩这些能力，并且如果后面想接入别的程序，还能通过一个独立服务端去同步和查询图片的元数据，于是于是就有了这个项目：**小曦 GitHub 图床 Broccoli V2.0 版本**。

设计这个工具的核心思路很简单：首先前端本体可以纯本地运行、图片实际上传到 GitHub 仓库、所有配置优先保存在浏览器本地（确保密钥安全），并且远程服务是可选增强，而不是强依赖。

---

# 这个项目解决了什么问题！

很多 GitHub 图床项目都有几个常见问题：

1. 只能上传，不能真正管理仓库和文件  
2. 配置很散，改个 CDN 或目录很麻烦  
3. 缺少图片处理能力，比如水印、压缩、命名规则  
4. 要么完全没有后端，要么后端绑得太死  
5. 接口能力不足，没法给别的程序复用

这个项目想解决的，就是把这些能力放进一个统一、清晰、可长期使用的工具里。
